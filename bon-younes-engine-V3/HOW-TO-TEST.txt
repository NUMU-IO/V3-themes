─── COMMON GOTCHAS ───────────────────────────────────────────────

• "EACCES on port 3000"
    Windows reserves some ports. Storefront is now on 3100. If 3100
    is also reserved, run:
      net stop winnat
      netsh int ip reset
      net start winnat
    Or pick another port in numu-storefront/package.json AND update
    numo-merchant-hub/.env.local's VITE_STOREFRONT_URL to match.

• "ENOTFOUND api.numu.io"
    CLI is hitting prod. Make sure ~/.numurc has:
      "api_url": "http://localhost:8000/api/v1"
    (already set today)

• "Loading preview…" stuck in editor
    Two things to check:
      (a) numo-merchant-hub/.env.local has:
            VITE_STOREFRONT_URL=http://{subdomain}.localhost:3100
      (b) The storefront on 3100 actually started (Terminal 3).

• Token expired (30-minute access tokens)
    Run `numu-theme login` again. Token refreshes in ~/.numurc.

• Port 5173 already in use
    You probably have `npm run dev` running in the theme folder
    AND `numu-theme dev` trying to start vite too. Pick one.


─── HOW DO I MAKE THEMES APPEAR IN THE MARKETPLACE? ──────────────

What you see today: "Currently active" + "Browse themes" lists the
EXTERNAL theme registered for YOUR store. The "Theme marketplace"
section at the bottom is the storefront for FREE published themes
that any merchant can install with one click.

Three concepts to keep separate:

  1. DEV-MODE EXTERNAL — what we've been doing.
     One store, one external theme, points at localhost:5173.
     Set via `numu-theme dev --store <id>`. Stays on that one store.
     Disappears when the store unsets it.

  2. INSTALLED EXTERNAL — a built bundle uploaded to NUMU storage.
     Belongs to one store. Permanent until the merchant uninstalls.
     Set via `numu-theme submit` after `numu-theme build`.

  3. MARKETPLACE THEME — a published theme that ANY merchant can
     browse + install in one click. This is what appears in the
     "Theme marketplace" panel at the bottom of the dashboard.

To make Bon Younes appear in the marketplace for free:

  STEP A — Build the production bundle
    cd C:\Users\PC\Desktop\NUMU\v3-themes\bon-younes-engine-V3
    node C:\Users\PC\Desktop\NUMU\numu-theme-cli\dist\index.js build

  STEP B — Submit to the marketplace
    node C:\Users\PC\Desktop\NUMU\numu-theme-cli\dist\index.js submit `
      --theme-id <new-uuid-or-omit-to-create>

    The first submit creates a theme record + uploads the bundle as
    version 0.1.0. Subsequent submits create new version rows.

  STEP C — Approve + publish in admin panel (numu-admin)
    Open http://localhost:5000 (numu-admin runs on port 5000).
    Find the new theme submission, set status = "approved",
    set pricing = "free", click Publish.

    This flips a flag in the DB so the public marketplace endpoint
    (`/api/v1/marketplace/themes`) returns it and merchants see it
    in the dashboard's "Theme marketplace" panel.

  STEP D — Repeat for each theme you want available.
    Do the same `build` + `submit` for:
      - luxury-minimal-engine-V3
      - vionne-engine-V3
      - empire-engine-V3
      - ...etc. for every theme in v3-themes/

    Then approve them all in numu-admin.

After that, when a merchant opens their dashboard and clicks any
theme in the marketplace panel, the platform pulls the right bundle
from NUMU's storage and renders it in their storefront. You can
keep adding bon-younes-v2, bon-younes-iced-edition, etc. — each is
its own marketplace row.
